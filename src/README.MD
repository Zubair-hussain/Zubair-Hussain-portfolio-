# Src Folder - Source Code Documentation

This folder contains the **source code** of **Zubair Hussain Portfolio** — a modern full-stack development project.

## 📁 Folder Structure

```bash
src/
├── app/              # Main application pages and routing
├── components/       # Reusable UI components
├── hooks/            # Custom React hooks
├── i18n/             # Internationalization (multi-language support)
├── lib/              # Utility functions, API calls, and helpers
├── messages/         # Translation files (for i18n)
├── styles/           # Global styles, CSS, or CSS modules
└── README.md
```

📋 Detailed Overview
app/

Contains main pages and routing logic
Entry point for the application (likely using Next.js App Router or similar)

components/

All reusable React components (Header, Footer, ProjectCard, AboutSection, etc.)
Keeps the codebase clean and maintainable

hooks/

Custom React hooks for logic reuse (e.g., useTheme, useScroll, useAnimation, etc.)

i18n/

Internationalization setup
Supports multiple languages for broader audience reach

messages/

JSON files containing translation strings for different languages

lib/

Utility functions, API integrations, data fetching logic, and helper functions
Core business logic and reusable utilities

styles/

Global CSS, themes, animations, and styling files
Keeps all styling organized in one place

🛠️ Tech Stack Highlights

Frontend: React / Next.js (Full-stack capable)
Styling: CSS / Tailwind / CSS Modules (based on your styles folder)
Internationalization: i18n setup with translation files
Full-Stack Ready: Backend logic can be integrated via lib/ or API routes

✅ Best Practices Applied

Clean and scalable folder structure
Separation of concerns (UI, logic, styles, translations)
Reusable components and hooks
Multi-language support ready
Well-organized for easy maintenance and future expansion

🚀 How to Work with Src

All development happens inside the src folder.
Components are built to be modular and reusable.
Styling is centralized in the styles directory.
Add new pages inside app/.
Create new custom hooks in hooks/ when needed.
Update translations in messages/ folder.
